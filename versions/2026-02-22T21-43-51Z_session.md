## Changes Made This Session

### 1. Nav Logo Image
Added optional `image` field to `data/nav.js`. When set to an image path, a logo renders next to the nav brand name.
- Configurable via `data/nav.js` → `image: "data/assets/logo.png"` (set to `null` to hide)
- Circular with `rounded-full`, fits nav background, sized 32×32px

### 2. Browser Tab Favicon
Added `<link rel="icon">` pointing to `data/assets/logo_black.png` so the logo appears in the browser tab next to the page title.

### 3. Dark / Light Mode
Full theme toggle system implemented:
- CSS custom properties (`:root` dark defaults, `html[data-theme="light"]` light overrides) in `globals.css`
- Semantic Tailwind color tokens (`skin-base`, `skin-text`, `skin-sub`, etc.) mapped to CSS variables
- **ThemeToggle** button (`●`/`○`) in the navbar — switches between dark and light palette
- Theme choice persists across page reloads via `localStorage`
- No flash-of-wrong-theme on load (anti-FOUC inline script in `<head>`)
- Primary CTA buttons invert correctly in both modes
- Custom cursor, particles, scrollbar, and all utility classes (`skill-tag`, `etag`, `ghlink`, etc.) are fully theme-aware

**Light mode palette:** warm cream backgrounds (`#f5f3ef`, `#edebe6`, `#e6e4df`), near-black text (`#0a0a0a`)
**Dark mode palette:** near-black backgrounds (`#080808`, `#0f0f0f`, `#161616`), warm cream text (`#f0ede6`)

### 4. Mobile Responsiveness
- **Hamburger menu** (`☰`) added to nav — visible on screens below `md` (768px)
- Opens a full-screen overlay with all nav links; closes on link click or `✕` button
- Locks body scroll while menu is open
- Nav padding, font sizes, section padding, and hero layout all reduce gracefully on mobile (`px-6 md:px-14`, `py-16 md:py-24`)
- Hero heading uses `clamp()` for fluid sizing down to small screens
- Large decorative background text hidden on small screens (`hidden sm:block`)
- CTA buttons and stats row reflow to full width on mobile

## Key Architecture Notes
- All color values are now CSS custom properties — editing theme colors only requires changing variables in `globals.css`
- To add new theme-aware colors, add to `:root` + `html[data-theme="light"]` in `globals.css`, then extend `tailwind.config` in `index.html` under `skin`
- Mobile menu is a React component (`MobileMenu`) rendered as a sibling to `<nav>` inside the `Nav` function
